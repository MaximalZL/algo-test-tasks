# Пример задач

# 1. Побег из лаборатории

## Условие

В лаборатории есть карта размера `n × m`.

На карте:

```text
S — стартовая позиция студента
E — выход
. — свободная клетка
# — стена
```

За один ход можно перейти в соседнюю клетку по стороне.

Нужно найти минимальное количество ходов от `S` до `E`.

Если добраться до выхода невозможно, вывести `-1`.

## Формат входных данных

```text
n m
карта
```

## Формат выходных данных

Одно число — минимальное количество ходов или `-1`.

## Ограничения

```text
1 ≤ n, m ≤ 1000
```

## Пример

**Ввод:**

```text
5 6
S..#..
.#.#..
.#....
.####.
....E.
```

**Вывод:**

```text
9
```

# 2. Секретный архив

## Условие

Хакер нашёл строку, состоящую из символов:

```text
( ) [ ] { }
```

Нужно проверить, является ли строка корректной скобочной последовательностью.

Последовательность считается корректной, если:

* каждой открывающей скобке соответствует закрывающая;
* скобки закрываются в правильном порядке.

## Формат входных данных

Одна строка `s`.

## Формат выходных данных

Вывести:

```text
YES
```

если последовательность корректная, иначе:

```text
NO
```

## Ограничения

```text
1 ≤ |s| ≤ 10^6
```

## Пример 1

**Ввод:**

```text
([]{})
```

**Вывод:**

```text
YES
```

## Пример 2

**Ввод:**

```text
([)]
```

**Вывод:**

```text
NO
```

# 3. Подозрительные фрагменты

## Условие

Дан массив из `n` целых чисел и число `k`.

Нужно найти количество подотрезков массива, сумма элементов на которых равна `k`.

## Формат входных данных

```text
n k
a1 a2 a3 ... an
```

## Формат выходных данных

Одно число — количество подходящих подотрезков.

## Ограничения

```text
1 ≤ n ≤ 200000
-10^9 ≤ ai ≤ 10^9
-10^14 ≤ k ≤ 10^14
```

## Пример

**Ввод:**

```text
5 3
1 2 1 2 1
```

**Вывод:**

```text
4
```

## Пояснение

Подходящие подотрезки:

```text
[1, 2]
[2, 1]
[1, 2]
[2, 1]
```

# 4. Сервер под нагрузкой

## Условие

На сервер приходят запросы. Для каждого запроса известно время его поступления в секундах.

Сервер считается перегруженным, если за последние `t` секунд пришло больше `k` запросов.

Даны времена поступления `n` запросов в неубывающем порядке.

Нужно определить, был ли сервер перегружен хотя бы один раз.

## Формат входных данных

```text
n t k
a1 a2 a3 ... an
```

где `ai` — время поступления `i`-го запроса.

## Формат выходных данных

Вывести:

```text
YES
```

если сервер был перегружен, иначе:

```text
NO
```

## Ограничения

```text
1 ≤ n ≤ 200000
1 ≤ t ≤ 10^9
1 ≤ k ≤ 200000
0 ≤ ai ≤ 10^9
```

## Пример

**Ввод:**

```text
6 10 3
1 3 7 12 13 14
```

**Вывод:**

```text
YES
```

## Пояснение

В промежутке с `7` по `14` включительно пришло 4 запроса, это больше `k = 3`.

# 5. Взлом пароля

## Условие

Пароль состоит из `n` символов. Для взлома каждого символа требуется некоторое количество энергии.

Дан массив `a`, где `ai` — энергия, необходимая для взлома `i`-го символа.

Хакер может выбрать число `x` — максимальную энергию, которую он готов потратить на один символ.

Все символы с `ai ≤ x` будут взломаны.

Нужно найти минимальное значение `x`, при котором будет взломано хотя бы `k` символов.

## Формат входных данных

```text
n k
a1 a2 a3 ... an
```

## Формат выходных данных

Одно число — минимальное подходящее значение `x`.

## Ограничения

```text
1 ≤ k ≤ n ≤ 200000
1 ≤ ai ≤ 10^9
```

## Пример

**Ввод:**

```text
7 4
8 3 10 2 5 7 1
```

**Вывод:**

```text
5
```

## Пояснение

При `x = 5` можно взломать символы с энергией:

```text
3, 2, 5, 1
```

Их ровно 4. Меньшее значение `x` не подходит.
